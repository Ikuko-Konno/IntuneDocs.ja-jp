### <a name="change-in-enrollment-workflow-with-intune-company-portal-on-corporate-ios-devices-authenticating-with-setup-assistant----1927359---"></a>セットアップ アシスタントで認証する企業の iOS デバイスで Intune ポータル サイトで登録ワークフローの変更します。 <!-- 1927359 -->
セットアップを使用したときに Apple の業務用デバイスの登録方法 - Apple Configurator、Apple のビジネス マネージャー、Apple School Manager、または Apple Device Enrollment Program (DEP) のいずれかを使用して iOS デバイスの登録のためのワークフローでの今後の変更があります。認証のアシスタントです。 この変更は、ユーザー アフィニティとともに登録されているデバイスにのみ適用されます。

#### <a name="how-does-this-affect-me"></a>ユーザーへの影響
この変更が ~~3 月~~ 4 月にロールアウトされると、Azure portal での Intune の登録プロファイルは、デバイスの認証方法およびポータル サイト アプリを受信するかどうかを指定できるように更新されます。 上記の方法で iOS デバイスを登録するワークフローが改良があります。 

- ときに新しいデバイスを登録して、セットアップ アシスタントを使用した認証は、ポータル サイト アプリを自動的に展開するかどうかを選択することができます。 「デバイスを識別する」画面と、登録フローで「デバイスを確認します」画面、エンドユーザーは表示されなくなります。  
- Apple の業務用デバイスの登録方法のいずれかでセットアップ アシスタントを使用して既に登録されているデバイスで条件付きアクセスを有効にする場合にアクションを行う必要があります。 これらのデバイスにポータル サイトをプッシュする特定の xml でのアプリ構成ポリシーを構成する必要があります。 これを行う手順は、追加情報のリンクでブログの投稿です。 この方法でポータル サイトをプッシュする場合は、エンドユーザーは「デバイスを識別する」画面と、登録フローで"デバイスの確認 画面に表示されなくなります。 
- 後でポータル サイトを展開していない場合、この変更はロールバック上記のように、アプリ構成プロファイルとエンドユーザーのダウンロード、アプリからポータル サイト アプリの保存、サインインすることができますは、エラー メッセージを取得します。 条件付きアクセス用アプリを使用することができません。 

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>この変更に対して必要な準備
変更後のワークフローを使用する場合は、ことを示す、エンド ユーザー ガイダンスを更新する必要があります。

- エンドユーザーでは、上記で説明した登録フローで 2 つの画面が表示されなくなります。 
- 自動的にデプロイされているときに、ポータル サイトにサインインし、app store からダウンロードしない必要があります。 

この変更の準備として、必要な場合は、アプリ構成ポリシーを今すぐ作成できます。 この新しいワークフローがロールアウト時に、コンソールで更新された登録プロファイルを確認します。 このロールアウト メッセージ センターでのお知らせします。 その後、セットアップ アシスタントで認証することで、エンドユーザーが DEP を使用を登録し、会社のポータルを使用して、条件付きアクセスの操作を実行する必要があります。

### <a name="plan-for-change-upcoming-fix-for-windows-10-email-profiles-in-intune---3904031--"></a>変更の計画: Intune で電子メール プロファイルを Windows 10 の今後の修正 <!--3904031-->
Intune は、4 月に Windows 10 用のプロファイルを更新したり、電子メール プロファイルが将来のバージョンの Windows 10 を使用し続けるようにもバグの修正に Intune サービスに電子メールを書き込む方法を更新しています。 この修正プログラムの展開後に実行する必要があるアクションがあります。

#### <a name="how-does-this-affect-me"></a>ユーザーへの影響
この変更に影響するで Windows 10 の電子メール プロファイルを使用する場合
- Windows 10 デスクトップ上のネイティブ メール クライアント、または
- Windows 10 Mobile の Outlook の電子メール クライアント

これにより、両方 Intune スタンドアロンとハイブリッド モバイル デバイス管理 (MDM) の顧客が影響します。

4 月の更新プログラムがロールアウト後は、(ハイブリッド MDM を使用している場合は、Configuration Manager 管理者コンソール) での Intune コンソールでこれらのプロファイルを再作成する必要があります。

アクションを実行しない場合は、4 月の更新プログラムの前に作成されたプロファイルを確認しますは以下です。

- 既存の電子メール プロファイルは Intune コンソールまたは Configuration Manager 管理者コンソール内でエラー状態に表示されますが、エンドユーザーには、電子メールへのアクセスがあります。 ただし、今後の Windows 更新プログラムがロールアウト後にこれらのプロファイルは機能しません。 これらのプロファイルの対象とするデバイスのエンドユーザーは、電子メールにアクセスをできなくなります。
- 後で年 4 月は反映されず、これらのプロファイルに対して行った編集は、デバイスを対象とします。
- 選択的ワイプは、4 月に修正プログラムがロールアウト後でも、これらのプロファイルを削除するのには機能しません。

アクションを実行し、電子メール プロファイルを再作成する場合は、エンドユーザーが最初に、電子メール プロファイルが展開されるものと同様の手順を経由する必要があります。 自分の電子メールは、新しいプロファイルが適用される更新プログラムを受け入れない限りの同期からブロックされます。

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>この変更に対して必要な準備
4 月の更新プログラムと修正プログラムがロールアウト後にのみアクションを実行する必要があります。 連絡先にメッセージ センターを通じて、Intune プロファイルを再作成を開始できるように、この変更が運用時にします。

Intune で Windows 10 の電子メール プロファイルを使用する場合は、次の手順を実行する必要があります。

1. 既存の Win 10 プロファイル設定をキャプチャします。
2. 割り当てを解除や、既存のプロファイルを削除します。
3. キャプチャした設定を使用して新しいプロファイルを作成し、同じグループに新しいプロファイルを割り当てる

この変更の確認、ヘルプデスク、エンドユーザーに通知する必要があります。 エラーの詳細と、これらのプロファイルを再作成するための手順で追加情報のサポートのブログの投稿を参照してください。
